# Audit Mode

**For:** Evaluating existing pages, flows, or copy.

Load this pattern when you have something that exists and want it analyzed.

## Activate

```
Read agency/agents/optimizer.md
Read agency/spawn/audit.md
```

Then give the Optimizer what you want audited.

## What Happens

1. **Optimizer receives** your page/flow/copy
2. **Optimizer analyzes** using CRO frameworks
3. **Optimizer delivers** prioritized findings and recommendations

## Audit Types

### Page Audit
Full landing page or website page analysis.

```
## Page Audit Request

**URL or content:**
[Link or paste the content]

**Page goal:**
[What conversion are you optimizing for?]

**Current performance:**
[Conversion rate, bounce rate if known]

**Main concern:**
[What made you want to audit this?]
```

### Flow Audit
Multi-step process (signup, checkout, onboarding).

```
## Flow Audit Request

**Flow name:**
[What process]

**Steps:**
[List the steps or provide screenshots]

**Drop-off data:**
[Where are you losing people?]

**Goal:**
[What's the desired end state?]
```

### Copy Audit
Review existing copy for effectiveness.

```
## Copy Audit Request

**Content:**
[Paste the copy]

**Content type:**
[Page, email, ad, etc.]

**Goal:**
[What should this copy achieve?]

**Concern:**
[What's not working?]
```

## What You Get Back

### Quick Wins
Changes that can be made immediately with high impact.

### Critical Issues
Fundamental problems that need addressing.

### Test Recommendations
Hypotheses to validate with A/B tests.

### Priority Matrix
Issues ranked by impact and effort.

## Example

> "Audit this signup flow. Here are the 4 screens: [attached]. We're seeing 60% drop-off between email entry and account creation. Goal is trial signup. Main concern is the form feels long but we need all the fields for compliance."

Optimizer will identify the friction and propose solutions.
